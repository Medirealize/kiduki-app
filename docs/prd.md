# KiDuKi 製品要求仕様書（PRD）

## 1. プロジェクト概要

### 1.1 目的
診察前の患者の「沈黙（未言語化の思考）」をAIとの対話で言語化し、診察の質を向上させる。

### 1.2 核心原則
- **診断・医療判断は一切行わない**
- 「気づき」と「整理」に特化する
- 患者の自己理解を深め、医師への伝達を支援する

### 1.3 ターゲットユーザー
- **年齢層**: 40〜70歳の外来患者
- **UX要件**: シンプルで温かみのあるインターフェースが必須

---

## 2. 機能要件（MVP）

### 2.1 対話フロー
- **Q1**: 最初の問い（来院理由・気になっていることのきっかけ）
- **Q2**: 深掘り問い（症状・経過・生活への影響の整理）
- **Q3**: 確認・まとめの問い（伝えたいことの明確化）
- **SUM**: 対話の要約（診察メモとして出力）

### 2.2 必須機能
- AIとの対話による問いへの回答
- Q1 → Q2 → Q3 の順序制御とプロンプト連携
- 診察メモ（SUM）の生成とコピー機能
- 高齢者でも使いやすいUI（フォントサイズ・余白・導線）

### 2.3 非機能要件
- 医療情報は診断に用いない旨の明示
- セキュアな通信（HTTPS）
- 個人情報の取り扱い方針の明示

---

## 3. 技術スタック（MVP）

| 領域 | 技術 |
|------|------|
| Frontend | Next.js (App Router), Tailwind CSS, shadcn/ui |
| AI | OpenAI API (GPT-4o / gpt-4o-mini) |
| 状態管理 | React Hook Form（入力用） |
| デプロイ | Vercel (Serverless Functions) |

---

## 4. 制約・前提
- 本システムは医療機器ではない
- 医師の診察を代替するものではない
- 患者の「言語化の補助」に限定する

---
